# Codex 启动提示词（剁手辅助 / 零后端单机版 / GitHub Pages v3）

请先读取当前项目根目录中的 `AGENTS.md`，并严格按文档执行。不要擅自更改技术栈，不要引入后端，不要改成依赖服务器才能运行的方案。

## 项目定位
这是一个移动端优先的消费辅助网站“剁手辅助”，目标用户是“有钱但不舍得花”以及“不知道该花多少”的个人用户。

## 强约束
1. 零后端单机版
2. 不做登录注册
3. 不做云同步
4. 不做服务端 API
5. 不做服务端数据库
6. 数据全部本地优先
7. 主数据库使用 IndexedDB，并用 Dexie 封装
8. 网站最终部署到 GitHub Pages
9. 通过 GitHub Actions 自动构建和发布
10. 禁止使用 Vercel
11. 必须支持静态导出
12. 页面必须移动端优先
13. UI 不能是默认白模或后台模板风格
14. 必须有统一主题、统一 UI 规范、适度美观动画
15. 动画要克制、顺滑、统一，不能花哨干扰使用

## 固定技术栈
- Next.js（App Router）
- TypeScript
- Tailwind CSS
- shadcn/ui
- Motion for React（motion/react）
- Dexie（IndexedDB）
- Zustand
- Zod
- date-fns
- pnpm

## MVP 页面
1. 首页
2. 账本页
3. 决策页
4. 设置页

## MVP 功能
- 月预算设置
- 本月已花 / 剩余可花统计
- 快速记账（生活必需 / 娱乐）
- 剁手决策按钮
- 奶茶额度系统
- 本地持久化
- JSON 导出 / 导入
- GitHub Pages 静态导出兼容
- GitHub Actions 发布工作流

## 设计与体验要求
- 视觉关键词：多巴胺、轻松、有趣、统一、克制
- 适当使用 emoji、卡片分区、线条层次
- 页面首屏应清楚展示预算、已花、剩余、奶茶额度和主操作入口
- 高优先级动效场景：页面进入、抽屉弹出、卡片切换、决策结果出现、数字状态变化
- 必须支持 reduced motion 或至少提供降低动画等级的机制
- 文案要友好，不说教，不企业化，不冰冷

## GitHub Pages 适配要求
- 使用静态导出方案
- 生成适用于 GitHub Pages 的配置
- 处理仓库子路径部署问题
- 生成 `.github/workflows/deploy-pages.yml`
- 在 `README.md` 中写清楚本地运行和 GitHub Pages 发布步骤

## 你的工作方式
- 直接开始实现，不要等待确认
- 每完成一个阶段，说明：
  1. 做了什么
  2. 新增和修改了哪些文件
  3. 下一步做什么
- 如果某种写法不兼容静态导出，优先退回纯客户端实现
- 如果某个 UI 方案显得像后台模板，请主动换掉
- 如果动画太重，请主动收敛，保持轻量统一
